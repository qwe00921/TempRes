	default=base64 zlib
	xml
	base64 uncompressed (=base64_gzip?)	
	base64 gzip
	base64 zlib
	csv

------- 存储文件格式支持 ------- 
	.tmx
	.dat
	.txt
	.json
	.bin


-----------------------------------------
编码可读行比较
	csv > xml > base64

编程简易性比较
	csv > xml > base64

文件尺寸比较
	csv < xml < base64

---------------------------------------------------------------------------
结论：
tmx  + base64:	保存文件最小、可读性差、需要解码、不够简单。
tmx  + xml:	保存文件中等、可读性中等、编程简易性中等。
tmx  + csv:     保存文件中等、可读性最好、编程简易性中等。
json + csv:	保存文件最大、可读性中等、编程最简单。

拟采用方案：
json + csv:
可读性好、编程最简单，虽然文件大一点，但是只是多个1K~2K，可忽略.

--------- 图层的组织 --------- 

底图		图像层（不要用普通层，浪费<data>空间）
路径		普通层
动态物件	对象层（可携带自定义串）
装饰层		对象层（若有）（可携带自定义串）
扩展层		对象层（若有）（可携带自定义串）
